Protocol

How it works

SignalInk uses Frequency Shift Keying (FSK) — the same fundamental technique used in old dial-up modems — implemented entirely in your browser with zero dependencies.

01

Text → Binary

Each character is converted to its 8-bit ASCII value. "Hi" becomes 01001000 01101001.

02

Binary → Sound

A 0 bit plays a tone at 18 kHz. A 1 bit plays at 20 kHz. Both are above the threshold of human hearing.

03

Sync framing

Each character is wrapped with a start byte (11111111) and stop byte (00000000) so the receiver locks on precisely.

04

Sound → Text

The receiving mic samples audio continuously. An FFT analyser detects which frequency dominates each time slot and reconstructs the bits.

Practical notes
  • Works best within 1–2 metres between devices
  • Loud ambient noise (>60 dB) may cause decoding errors
  • Not all phone speakers reproduce 20 kHz — laptop speakers work best
  • Both devices must use the same frequency & bit-duration settings
  • No data ever leaves your device — there is no server
